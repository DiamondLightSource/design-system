# Documentation Guide

This repository contains the Diamond Design System website.

The website is the primary source of guidance for designing and building scientific software using the Diamond Design System at Diamond Light Source. Although developed for Diamond, its guidance should be applicable to other scientific facilities wherever practical.

The website complements Storybook by focusing on **why**, **when** and **how** to use the Design System. Storybook provides interactive documentation for implemented components. Content may evolve between the website and Storybook as the Design System matures.

## Audience

- Engineers
- Scientists
- Designers
- Product owners

Assume users have varying levels of familiarity with design systems.

## Goals

Documentation should:

- explain why, when and how to use the Design System
- be practical and task-oriented
- evolve alongside the Design System
- favour reusable guidance over isolated examples
- use scientific workflows and real-world examples where appropriate

## Technology

- Built with **SciReactUI**, Diamond's React component library based on MUI.
- Use SciReactUI components wherever possible.
- Use semantic design tokens and CSS variables.
- Support both light and dark modes.

## Principles

- Use British English.
- Keep guidance concise, practical and actionable.
- Explain the reasoning behind recommendations.
- Reuse and extend existing guidance where appropriate.
- Prefer existing patterns and components before introducing new ones.

## Design

- Calm, clean and editorial.
- Typography before decoration.
- Use icons sparingly to aid navigation.
- Prioritise accessibility and readability.
- Maintain consistency across the website.
